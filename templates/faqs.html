Frequently Asked Questions

 Navigate to the OpenCC application and just fill out the form with the correct information.
 You can make any number of requests as long as you have not already made the same request. Please do not make unnecessary requests as it will delay the search, notifications you are expecting and potentially delay others to get notified.
 As of now only the classes that are related to Candidacy (OTA, PTA, NURSING, RED TECH etc) are NOT supported. These classes are assigned by the department so even if a section is open you can't get in since you would permission from the department. If the application is still missing other classes except the ones mentioned above then please drop a message to Hamidur Rahman.
 You can do one of the following to stop getting text messages. Regardless of the choice you make, a text will be sent saying that your requested class(es) have been deleted. If you do not get a text message that means you did not have a request to be deleted.

1 .Completely Stop Getting Text Messages:

 You can simply reply to the message saying: cancel all. You CAN NOT write anything else before or after cancel all. It doesn't matter if the text cancel all is upper case or not as long as the replied message only contains cancel all (i.e you can still add requests and get notified even if you take this approach.)

2. Stop Getting Text Messages for Only One Class:

 You can simply reply to the message saying: cancel [5-digit-class-number] ex. cancel 45678. You can find the classes you requested in the chat. You CAN NOT write anything else before or after cancel [5-digit-class-number]. It does not matter if the text cancel [5-digit-class-number] is upper case or not.
 This is a convenient way to stop getting notifications as it allows you to stop getting notified for a class that you have found a seat for, if you decided you will not take this class, or if you decide to get notifications for the same course but a different section (i.e you can still add requests and get notified even if you take this approach.)
 This application has a scheduler which runs every 2.5 minutes. The scheduler gets all the entered requests and search each request(class). If any of the classes are open then those who have made the request for this class will get notified right away in a couple of seconds. Keep in mind that if CUNYfirst is down due to maintenance then the application will not be able to search requested class(es) until it is back online. If a class becomes open right before CUNYfirt goes under maintenance, no one will be able to get that class until CUNYfirst is back online. Since the scheduler runs every 2.5 minutes, you still will have a high chance of getting this class if CUNYfirst is back online after maintenance.
